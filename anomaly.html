Operational Anomaly
Detection

Methods Isolation Forest · LOF
Detection rates 12% (IF) · 4.5% (LOF)
Tools Python · scikit-learn · pandas
Application Revenue leakage · Fraud detection

The Problem

Revenue leakage and fraud represent a persistent operational risk for businesses managing large transaction volumes. Identifying anomalous patterns in high-dimensional datasets requires methods that go beyond rule-based detection — particularly when fraudulent behaviour adapts to circumvent known thresholds. Unsupervised anomaly detection provides a scalable, adaptive approach that doesn't require labelled fraud examples to function.

Methodology

An anomaly detection pipeline was built using two complementary unsupervised approaches, evaluated in parallel against the same customer dataset:

  • Isolation Forest: isolates anomalies by recursively partitioning the feature space, flagging observations that require fewer splits to isolate — achieving a 12% anomaly detection rate
  • Local Outlier Factor (LOF): identifies anomalies based on local density deviation relative to neighbours — achieving a 4.5% detection rate, reflecting higher specificity

Business-relevant indicators were engineered to improve interpretability of flagged cases, translating model outputs into operational language for customer support and revenue assurance teams.

Running both methods in parallel surfaces two distinct anomaly populations: high-volume outliers (captured by Isolation Forest) and localised density anomalies (captured by LOF) — together providing broader coverage than either method alone.

Commercial Implications

For a revenue assurance or fraud operations team, this pipeline provides an automated first-pass triage layer — surfacing the highest-risk transactions for human review rather than requiring manual inspection of entire datasets. The interpretability layer (business-relevant indicators) ensures flagged cases can be acted on without requiring the reviewing analyst to understand the underlying model mechanics.
